## Project Overview

Test::ValkeyServer is a Perl CPAN module that spawns and manages temporary valkey-server instances for use in test suites.
It uses Mouse for OOP, fork-based process management, and supports both Unix sockets (default) and TCP ports. Forked from Test::RedisServer for Valkey compatibility.

The main module is `lib/Test/ValkeyServer.pm`.

## Build and Test Commands

```bash
# Run full test suite (including release tests)
minil test --release
```

**Note:** `Build.PL` is auto-generated by Minilla — never edit it directly. Build config lives in `minil.toml`.

## Architecture

- **OOP Framework:** Mouse (lightweight Moose alternative) with `make_immutable`
- **Process Model:** `fork()` to spawn valkey-server; parent monitors readiness by scanning the log file for "Ready to accept connections" (case-insensitive regex for cross-version compatibility)
- **Lifecycle:** `BUILD` auto-starts by default; `DEMOLISH` ensures cleanup. Manual control via `start()`, `stop()`, `exec()`, `wait_exit()`
- **exec() method:** Designed for integration with Test::TCP — calls `exec` directly instead of forking. Not compatible with cluster mode (no opportunity to run `valkey-cli` post-exec)
- **Cluster mode:** `cluster => 1` creates a single-node cluster (Valkey 8.1+). Uses `valkey-cli --cluster create` after server starts. Requires TCP (auto-allocates port via `empty_port()` if not specified)
- **Cleanup:** Tracks `_owner_pid` to prevent child processes from running the destructor's cleanup logic
- **stop() shutdown:** Disables `appendonly` and `save` before shutdown to prevent server hang when tmpdir has already been removed
- **Client connection:** `_redis` client is created at `start()` time (not lazily) because the socket may disappear during shutdown. Uses `Redis.pm` (protocol-compatible with Valkey)
